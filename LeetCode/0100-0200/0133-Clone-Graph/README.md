# [133. Clone Graph](https://leetcode.com/problems/clone-graph/)

## Описание задачи

Вам дана ссылка на узел в связном неориентированном графе. Необходимо вернуть **глубокую копию** (клон) этого графа.

Каждый узел графа содержит значение `val` (целое число) и список `neighbors` (список соседних узлов).

_Глубокая копия означает, что вы должны создать абсолютно новые объекты узлов, а не просто скопировать ссылки на старые. Значения и структура связей должны быть полностью идентичны оригиналу._

**Пример 1:**

- **Input:** adjList = [[2,4],[1,3],[2,4],[1,3]]
- **Output:** [[2,4],[1,3],[2,4],[1,3]]
- **Пояснение:** Граф состоит из 4 узлов.
  Узел 1 соединен с узлами 2 и 4.
  Узел 2 соединен с узлами 1 и 3.
  И так далее. Возвращаемый граф должен быть новой копией с точно такой же структурой.

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Проблема глубокого копирования:** Нам нужно пройти по всему оригинальному графу и для каждого узла создать его клона. Проблема в том, что граф неориентированный: если узел А ссылается на Б, то Б ссылается на А. Если мы просто запустим рекурсию, мы уйдем в бесконечный цикл (А -> Б -> А -> Б...).
- **Отслеживание клонов (Hash Map):** Чтобы избежать зацикливания и не создавать один и тот же узел дважды, нам нужна память. Заведем хеш-таблицу `old_to_new`, где ключом будет оригинальный узел, а значением — его вновь созданный клон.
- **Обход в глубину (DFS):**
  1. Если текущий узел уже есть в нашей таблице `old_to_new`, значит, мы его уже клонировали. Просто возвращаем его клона из таблицы.
  2. Если узла нет, мы создаем его клона `copy_node = Node(node.val)`.
  3. Сразу же (!) добавляем его в таблицу `old_to_new`. Это критически важно сделать _до_ обхода соседей, иначе мы можем поймать цикл.
  4. Проходим по всем соседям оригинального узла, рекурсивно вызываем для них DFS и добавляем результаты в список `neighbors` нашего `copy_node`.

### 2. Алгоритм

1. Создать пустую хеш-таблицу `old_to_new = {}`.
2. Определить рекурсивную функцию `dfs(node)`:
   - Если `node` равен `None`, вернуть `None`.
   - Если `node` уже находится в `old_to_new`, вернуть `old_to_new[node]`.
   - Создать новый узел `copy = Node(node.val)`.
   - Сохранить его в таблицу: `old_to_new[node] = copy`.
   - Запустить цикл по всем соседям `neighbor` в `node.neighbors`:
     - Вызвать `dfs(neighbor)` и добавить возвращенный узел в `copy.neighbors`.
   - Вернуть `copy`.
3. Вернуть результат вызова `dfs(node)`, передав туда исходный узел.

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(V + E)**, где V — количество узлов (Vertices), а E — количество ребер (Edges). Мы посещаем каждый узел ровно один раз и проходим по каждому ребру.
- **Сложность по памяти (Space Complexity):** **O(V)**. Наша хеш-таблица `old_to_new` будет хранить ссылки на все V узлов графа. Также стек вызовов рекурсии в худшем случае может достигнуть глубины V (если граф вытянут в линию).
