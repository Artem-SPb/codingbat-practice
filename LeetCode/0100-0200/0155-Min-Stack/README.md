# 155. Min Stack

## Описание задачи

Разработайте стек, который поддерживает стандартные операции `push`, `pop`, `top`, а также извлечение минимального элемента `getMin` за константное время.

Реализуйте класс `MinStack`:

- `MinStack()` инициализирует объект стека.
- `void push(int val)` добавляет элемент `val` на вершину стека.
- `void pop()` удаляет элемент с вершины стека.
- `int top()` возвращает элемент на вершине стека.
- `int getMin()` возвращает минимальный элемент в стеке.

**Ограничение:** Вы должны реализовать решение с временной сложностью O(1) для каждой функции.

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Стандартный стек:** Операции `push`, `pop` и `top` в обычном массиве (или списке) и так выполняются за время O(1). Главная проблема — метод `getMin()`.
- **Поиск минимума:** Если мы будем просто перебирать стек при каждом вызове `getMin()`, сложность будет O(n), что нарушает условие задачи. Значит, нам нужно как-то "запоминать" минимум заранее.
- **Оптимальный подход:** Значение минимума в стеке изменяется только тогда, когда мы добавляем элемент меньше текущего минимума, или когда мы удаляем элемент, который был минимумом. Поскольку элементы удаляются строго в обратном порядке (LIFO), мы можем хранить минимальное значение на момент добавления каждого элемента.
- **Реализация:** Вместо того чтобы хранить в стеке просто числа, мы будем хранить пары (кортежи): `(значение, минимум_на_текущий_момент)`. Таким образом, верхний элемент стека всегда "знает", какой был абсолютный минимум во всем стеке на момент его добавления.

### 2. Алгоритм

1. В методе `init` создаем пустой массив `stack`.
2. При вызове `push(val)`:
   - Если стек пуст, текущий минимум равен самому `val`. Добавляем в стек `(val, val)`.
   - Если стек не пуст, берем минимум из верхнего элемента стека: `current_min = stack[-1][1]`. Вычисляем новый минимум: `new_min = min(val, current_min)`. Добавляем в стек: `stack.append((val, new_min))`.
3. При вызове `pop()` просто удаляем последний элемент из массива: `stack.pop()`.
4. При вызове `top()` возвращаем первое значение из верхнего кортежа: `stack[-1][0]`.
5. При вызове `getMin()` возвращаем второе значение из верхнего кортежа: `stack[-1][1]`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** O(1) для всех операций (`push`, `pop`, `top`, `getMin`). Никаких циклов не используется.
- **Сложность по памяти (Space Complexity):** O(n), так как мы храним дополнительную информацию (текущий минимум) для каждого из $n$ элементов стека.
