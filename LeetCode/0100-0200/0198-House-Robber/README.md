# [198. House Robber](https://leetcode.com/problems/house-robber/)

## Описание задачи

Вы — профессиональный грабитель, планирующий обчистить дома вдоль улицы. В каждом доме спрятана определенная сумма денег. Единственное ограничение, которое мешает вам ограбить все дома: соседние дома соединены системой безопасности. Если два соседних дома будут ограблены в одну и ту же ночь, сработает сигнализация и приедет полиция.

Вам дан целочисленный массив `nums`, представляющий сумму денег в каждом доме. Необходимо вернуть максимальную сумму денег, которую вы можете ограбить за ночь, не подняв тревогу.

**Пример 1:**

- **Input:** nums = [1,2,3,1]
- **Output:** 4
- **Пояснение:** Грабим дом 1 (деньги = 1) и дом 3 (деньги = 3). Итого: 1 + 3 = 4.

**Пример 2:**

- **Input:** nums = [2,7,9,3,1]
- **Output:** 12
- **Пояснение:** Грабим дом 1 (деньги = 2), дом 3 (деньги = 9) и дом 5 (деньги = 1). Итого: 2 + 9 + 1 = 12.

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Проблема выбора:** Возле каждого дома у нас есть два варианта:
  1. **Ограбить его:** тогда мы прибавляем деньги из этого дома к максимальной сумме, которую мы собрали два дома назад (ведь предыдущий дом грабить нельзя).
  2. **Пропустить его:** тогда у нас остается та же максимальная сумма, которую мы имели на предыдущем доме.
- **Перекрытие подзадач:** Если использовать обычную рекурсию, мы будем раз за разом пересчитывать одни и те же комбинации для одних и тех же домов. Это займет O(2^n) времени.
- **Динамическое программирование:** Нам нужно просто идти слева направо и на каждом шаге принимать оптимальное решение, опираясь на лучшие результаты прошлых шагов.
- Формула перехода состояния: `Текущий_максимум = max(Деньги_в_этом_доме + Максимум_через_один_дом, Максимум_на_предыдущем_доме)`.
- **Оптимизация памяти:** Чтобы посчитать результат для текущего дома, нам нужно знать значения только для **двух предыдущих** домов. Значит, нам не нужно хранить весь массив ответов, достаточно двух переменных (`rob1` и `rob2`).

### 2. Алгоритм

1. Инициализируем две переменные для хранения предыдущих состояний: `rob1 = 0` (максимум за два шага назад) и `rob2 = 0` (максимум на прошлом шаге).
2. Запускаем цикл по каждому числу `n` в массиве `nums`:
   - Вычисляем максимум для текущего шага: `current = max(n + rob1, rob2)`.
   - Сдвигаем наше "окно" вперед: `rob1 = rob2`, а `rob2 = current`.
3. После завершения цикла в `rob2` будет лежать максимальная возможная сумма для всей улицы. Возвращаем `rob2`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N)**, где N — количество домов. Мы проходим по массиву ровно один раз.
- **Сложность по памяти (Space Complexity):** **O(1)**. Мы используем только две дополнительные переменные для отслеживания состояний, независимо от размера входного массива.
