# [150. Evaluate Reverse Polish Notation](https://leetcode.com/problems/evaluate-reverse-polish-notation/)

## Описание задачи

Вам дан массив строк `tokens`, который представляет собой арифметическое выражение в **Обратной польской записи (Reverse Polish Notation)**.

Необходимо вычислить это выражение и вернуть целочисленный результат.

**Особенности и ограничения:**

- Допустимые операторы: `+`, `-`, `*`, `/`.
- Каждый операнд может быть как целым числом, так и другим выражением.
- Деление между двумя целыми числами должно **усекаться к нулю** (то есть отбрасывать дробную часть).
- Гарантируется, что заданное выражение всегда валидно (не будет деления на ноль).

**Пример 1:**

- **Input:** tokens = ["2","1","+","3","*"]
- **Output:** 9
- **Пояснение:** ((2 + 1) \* 3) = 9

**Пример 2:**

- **Input:** tokens = ["4","13","5","/","+"]
- **Output:** 6
- **Пояснение:** (4 + (13 / 5)) = (4 + 2) = 6

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Специфика ОПЗ:** Главное преимущество обратной польской записи в том, что ей не нужны скобки. Мы читаем выражение слева направо. Как только нам встречается число — мы должны его запомнить. Как только встречается знак операции — мы должны применить его к **двум последним запомненным числам**.
- **Выбор структуры данных:** Принцип "последним пришел — первым обработан" (LIFO) однозначно указывает на **Стек**.
- **Алгоритм обработки:**
  1. Идем по списку токенов слева направо.
  2. Если токен — это число, преобразуем его в `int` и кладем на вершину стека (`push`).
  3. Если токен — это оператор (`+`, `-`, `*`, `/`), мы извлекаем два верхних числа из стека (`pop`). _Важно:_ первое извлеченное число будет правым операндом, а второе — левым (например, для токенов `["5", "2", "-"]` мы должны посчитать `5 - 2`, а не `2 - 5`).
  4. Вычисляем результат операции и кладем его обратно в стек.
  5. В самом конце в стеке останется ровно одно число — это и есть ответ.

- **Подводный камень (Python):** В условии сказано, что деление усекается _к нулю_. В Python обычный оператор целочисленного деления `//` округляет _вниз_ (к минус бесконечности). Для положительных чисел `6 // 4 == 1` (верно), но для отрицательных `-6 // 4 == -2` (а нам нужно `-1`). Поэтому безопаснее использовать конструкцию `int(a / b)`.

### 2. Алгоритм

1. Инициализировать пустой список `stack`.
2. Запустить цикл по каждому `token` в `tokens`:
   - Если `token` равен `"+"`: делаем `stack.append(stack.pop() + stack.pop())`.
   - Если `token` равен `"-"`: извлекаем `b = stack.pop()`, `a = stack.pop()`, делаем `stack.append(a - b)`.
   - Если `token` равен `"*"`: делаем `stack.append(stack.pop() * stack.pop())`.
   - Если `token` равен `"/"`: извлекаем `b = stack.pop()`, `a = stack.pop()`, делаем `stack.append(int(a / b))`.
   - Иначе (это число): делаем `stack.append(int(token))`.
3. Вернуть `stack[0]`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N)**, где N — количество элементов в массиве `tokens`. Мы проходим по списку ровно один раз, все операции со стеком (`append` и `pop`) выполняются за **O(1)**.
- **Сложность по памяти (Space Complexity):** **O(N)**. В худшем случае (если выражение состоит сначала из одних чисел, а потом из одних операторов) стек будет хранить до `N/2` чисел одновременно.
