# [128. Longest Consecutive Sequence](https://leetcode.com/problems/longest-consecutive-sequence/)

## Описание задачи

Дан неотсортированный массив целых чисел `nums`. Необходимо вернуть длину самой длинной последовательности последовательных элементов (элементов, идущих друг за другом: 1, 2, 3, 4...).

**Строгое ограничение:** Алгоритм должен работать за время **O(N)**.

**Пример 1:**

- **Input:** nums = [100, 4, 200, 1, 3, 2]
- **Output:** 4
- **Пояснение:** Самая длинная последовательность — это `[1, 2, 3, 4]`. Её длина равна 4. (Числа 100 и 200 идут отдельно и не образуют длинных цепочек).

**Пример 2:**

- **Input:** nums = [0, 3, 7, 2, 5, 8, 4, 6, 0, 1]
- **Output:** 9

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Проблема сортировки:** Первое, что приходит в голову — отсортировать массив. Тогда мы сможем просто пройти по нему один раз и посчитать длину цепочки. Но сортировка занимает `O(N log N)` времени, а условие требует `O(N)`. Значит, сортировать нельзя.
- **Быстрый поиск (Hash Set):** Нам нужно как-то мгновенно узнавать, есть ли следующее число в массиве. Преобразуем наш массив во множество (`set`). Поиск элемента в множестве занимает `O(1)`.
- **Оптимизация обхода:** Если мы будем проверять цепочку для _каждого_ числа, мы сделаем много лишней работы. Например, для массива `[1, 2, 3, 4]`, если мы начнем проверять с `2` (найдем 2, 3, 4), потом с `3` (найдем 3, 4) — это будет медленно.
- **Главный инсайт:** Нам нужно начинать отсчет только с **начала последовательности**! Как понять, что число является началом? Очень просто: у него нет левого соседа. То есть, если числа `num - 1` нет в нашем множестве, значит `num` — это начало новой цепочки.

### 2. Алгоритм

1. Преобразовать массив `nums` в множество `num_set` для поиска за `O(1)`.
2. Инициализировать переменную `longest_streak = 0`.
3. Пройти циклом по каждому числу `num` в `num_set`:
   - **Проверка на начало:** Если `num - 1` НЕ находится в `num_set`, значит это начало новой последовательности.
   - Если это начало, запускаем внутренний цикл: пока `num + 1` существует в `num_set`, увеличиваем длину текущей последовательности и переходим к следующему числу.
   - Обновляем `longest_streak`, выбирая максимум между текущим рекордом и длиной только что найденной последовательности.
4. Вернуть `longest_streak`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N)**. Создание множества занимает `O(N)`. Несмотря на наличие вложенного цикла (цикл `while` внутри `for`), каждое число из массива посещается внутренним циклом не более одного раза. В итоге мы делаем линейное количество операций.
- **Сложность по памяти (Space Complexity):** **O(N)**. Мы создаем множество `num_set`, которое в худшем случае содержит все уникальные элементы исходного массива.
