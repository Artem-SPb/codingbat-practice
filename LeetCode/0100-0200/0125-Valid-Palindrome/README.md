# [125. Valid Palindrome](https://leetcode.com/problems/valid-palindrome/)

## Описание задачи

Дана строка `s`. Необходимо вернуть `true`, если она является палиндромом, и `false` в противном случае.

При проверке необходимо сначала удалить все не-буквенно-цифровые символы и привести все заглавные буквы к нижнему регистру.

**Пример 1:**
**Input:** s = "A man, a plan, a canal: Panama"
**Output:** true
**Пояснение:** После обработки строка становится "amanaplanacanalpanama", что является палиндромом.

**Пример 2:**
**Input:** s = "race a car"
**Output:** false
**Пояснение:** После обработки строка становится "raceacar", что не является палиндромом.

---

## Мой подход к решению

### 1. Размышления (Thought Process)

Задача состоит из двух частей:

1.  **Очистка строки:** Нужно отфильтровать все ненужные символы (пробелы, знаки препинания) и привести строку к единому регистру.
2.  **Проверка на палиндром:** Нужно проверить, читается ли очищенная строка одинаково в обоих направлениях.

Самый эффективный алгоритм для проверки на палиндром — это **"Два указателя" (Two Pointers)**.

- **Идея:** Мы ставим один указатель (`left` или `l`) в начало строки, а второй (`right` или `r`) — в конец.
- **Движение:** На каждом шаге мы сравниваем символы, на которые указывают `l` и `r`. Если они совпадают, мы сдвигаем оба указателя навстречу друг другу (`l` вправо, `r` влево).
- **Условие остановки:** Если на каком-то шаге символы не совпали, значит, это не палиндром, и мы сразу возвращаем `False`. Если указатели встретились или пересеклись (`l >= r`), а мы так и не нашли несовпадений, значит, это палиндром, и мы возвращаем `True`.

**Оптимизация:** Можно объединить очистку и проверку в один цикл. Вместо того чтобы создавать новую строку, мы можем двигать указатели по исходной строке, просто "перепрыгивая" через ненужные символы. Это экономит память (O(1) вместо O(n)).

### 2. Алгоритм (Оптимальный, с O(1) по памяти)

1.  Инициализировать два указателя: `l = 0` (начало строки) и `r = len(s) - 1` (конец строки).
2.  Запустить цикл `while l < r`.
3.  Внутри цикла:
    a. Сдвигать левый указатель `l` вправо, пока он не укажет на буквенно-цифровой символ (и пока `l < r`).
    b. Сдвигать правый указатель `r` влево, пока он не укажет на буквенно-цифровой символ (и пока `l < r`).
    c. Сравнить символы `s[l]` и `s[r]`, предварительно приведя их к нижнему регистру.
    d. Если они не равны, немедленно вернуть `False`.
    e. Если равны, сдвинуть оба указателя: `l += 1`, `r -= 1`.
4.  Если цикл завершился, значит, все проверки пройдены успешно. Вернуть `True`.

---

## Код решения

```python
class Solution:
    def isPalindrome(self, s: str) -> bool:
        l, r = 0, len(s) - 1

        while l < r:
            # Пропускаем не-буквенно-цифровые символы слева
            while l < r and not s[l].isalnum():
                l += 1

            # Пропускаем не-буквенно-цифровые символы справа
            while l < r and not s[r].isalnum():
                r -= 1

            # Сравниваем символы
            if s[l].lower() != s[r].lower():
                return False

            # Сдвигаем указатели
            l += 1
            r -= 1

        return True
```

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(n)**, где `n` — длина строки. Каждый указатель проходит по строке не более одного раза.
- **Сложность по памяти (Space Complexity):** **O(1)**. Мы не используем дополнительную память, размер которой зависит от размера входных данных. Все операции происходят "на месте" (in-place).

---

## Ключевые выводы / Паттерн

- **Два указателя (Two Pointers):** Этот паттерн чрезвычайно полезен для задач, где нужно анализировать массив/строку с обоих концов одновременно. Указатели, движущиеся навстречу друг другу, — классический способ проверки на палиндром, поиска пар элементов и т.д.
- **In-place операции:** Решение, которое избегает создания новых больших структур данных (как новая "очищенная" строка), часто считается более качественным и эффективным, особенно если память является критическим ресурсом.
