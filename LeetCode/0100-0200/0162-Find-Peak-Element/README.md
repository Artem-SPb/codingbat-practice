# [162. Find Peak Element](https://leetcode.com/problems/find-peak-element/)

## Описание задачи

Пиковый элемент — это элемент, который строго больше своих соседей.

Вам дан целочисленный массив `nums` (где соседние элементы не равны друг другу: `nums[i] != nums[i+1]`). Найдите пиковый элемент и верните его индекс. Если в массиве несколько пиков, верните индекс **любого** из них.

Вы можете представить, что по краям массива находятся бесконечно малые числа (`nums[-1] = -∞` и `nums[n] = -∞`). Другими словами, элемент на краю считается пиком, если он больше своего единственного соседа.

**Строгое ограничение:** Вы должны написать алгоритм со сложностью по времени **O(log N)**.

**Пример 1:**

- **Input:** nums = [1,2,3,1]
- **Output:** 2
- **Пояснение:** Элемент 3 является пиком (больше 2 и больше 1). Его индекс — 2.

**Пример 2:**

- **Input:** nums = [1,2,1,3,5,6,4]
- **Output:** 5
- **Пояснение:** В массиве два пика: элемент 2 (индекс 1) и элемент 6 (индекс 5). Вернуть можно любой из этих индексов.

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Как искать в неотсортированном массиве?** Поскольку по краям у нас пропасти (`-∞`), любой подъем графика гарантированно приведет к пику. График не может расти бесконечно, потому что в конце массива он обязан упасть в пропасть.
- **Ищем по наклону (Градиентный спуск наоборот):** Берем случайную точку посередине. Смотрим на её правого соседа:
  1. Если сосед справа **больше** (`nums[mid] < nums[mid + 1]`), значит, мы находимся на подъеме. Пик точно находится где-то **справа**. Мы можем смело отбросить всю левую половину.
  2. Если сосед справа **меньше** (`nums[mid] > nums[mid + 1]`), значит, мы находимся на спуске (или на самом пике). Пик точно находится **слева** (включая саму точку `mid`). Мы отбрасываем правую половину.
- **Схлопывание границ:** Как и в задаче 153 (поиск минимума), мы продолжаем сжимать границы поиска, пока они не сойдутся в одной точке. Эта точка и будет пиком.

### 2. Алгоритм

1. Инициализировать указатели: `left = 0`, `right = len(nums) - 1`.
2. Пока `left < right`:
   - Вычислить середину: `mid = (left + right) // 2`.
   - Если `nums[mid] < nums[mid + 1]` (график идет вверх):
     - Сдвинуть `left = mid + 1`.
   - Иначе (график идет вниз):
     - Сдвинуть `right = mid`.
3. Когда цикл завершится, `left` будет равен `right`, и оба будут указывать на индекс пикового элемента. Вернуть `left`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(log N)**. Мы отбрасываем половину оставшегося массива на каждой итерации.
- **Сложность по памяти (Space Complexity):** **O(1)**. Мы используем только переменные для хранения индексов `left`, `right` и `mid`.
