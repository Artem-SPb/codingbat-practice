# [141. Linked List Cycle](https://leetcode.com/problems/linked-list-cycle/)

## Описание задачи

Дан заголовок `head` односвязного списка. Необходимо определить, содержит ли список цикл (зацикливание).

Цикл в связном списке существует, если в нём есть узел, в который можно повторно попасть, непрерывно переходя по ссылке `next`.

Вернуть `true`, если в списке есть цикл. В противном случае вернуть `false`.

**Пример 1:**

- **Input:** head = [3,2,0,-4], pos = 1 (узел -4 ссылается обратно на узел 2)
- **Output:** true

**Пример 2:**

- **Input:** head = [1,2], pos = 0 (узел 2 ссылается обратно на узел 1)
- **Output:** true

**Пример 3:**

- **Input:** head = [1], pos = -1 (цикла нет)
- **Output:** false

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Подход с дополнительной памятью (Hash Set):** Самое простое решение — идти по списку и складывать посещенные узлы (ссылки на объекты в памяти) в множество `set`. Если очередной узел уже есть в множестве — значит, мы зашли на второй круг (цикл есть). Если дошли до `None` — цикла нет. Сложность по времени O(N), но по памяти тоже O(N).
- **Оптимальный подход (Floyd's Tortoise and Hare):** Представим беговую дорожку на стадионе. Если два бегуна стартуют одновременно, но один бежит строго в два раза быстрее другого:
  1. Если дорожка прямая (цикла нет), быстрый бегун просто добежит до финиша (`None`).
  2. Если дорожка замкнута в круг (цикл есть), быстрый бегун рано или поздно обогнет круг и догонит медленного со спины.
- **Реализация:** Заводим два указателя: `slow` (делает 1 шаг за раз: `curr.next`) и `fast` (делает 2 шага за раз: `curr.next.next`). Если они когда-нибудь укажут на один и тот же объект в памяти (`slow == fast`), значит, в списке гарантированно есть зацикливание.

### 2. Алгоритм

1. Инициализировать два указателя на старте: `slow = head`, `fast = head`.
2. Запустить цикл, пока `fast` и `fast.next` не равны `None`:
   - Сдвинуть медленный указатель на 1 шаг: `slow = slow.next`.
   - Сдвинуть быстрый указатель на 2 шага: `fast = fast.next.next`.
   - Если `slow == fast`, вернуть `True`.
3. Если цикл завершился (быстрый указатель уперся в конец списка), вернуть `False`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N)**, где N — общее количество узлов. Если цикла нет, быстрый указатель дойдет до конца за $N/2$ итераций. Если цикл есть, указатели встретятся внутри кольца менее чем за N шагов.
- **Сложность по памяти (Space Complexity):** **O(1)**. Мы используем ровно две переменные-ссылки (`slow` и `fast`), затраты памяти константны.
