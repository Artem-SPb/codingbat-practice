# [102. Binary Tree Level Order Traversal](https://leetcode.com/problems/binary-tree-level-order-traversal/)

## Описание задачи

Дан корень бинарного дерева `root`. Необходимо вернуть обход значений его узлов по уровням (то есть слева направо, уровень за уровнем).

**Пример 1:**

    3

/ \
 9 20
/ \
 15 7

- **Input:** root = [3,9,20,null,null,15,7]
- **Output:** [[3],[9,20],[15,7]]

**Пример 2:**

- **Input:** root = [1]
- **Output:** [[1]]

**Пример 3:**

- **Input:** root = []
- **Output:** []

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Выбор алгоритма:** Стандартная рекурсия (DFS — обход в глубину) стремится сначала пройти по ветке до самого низа. Нам же нужно двигаться строго горизонтально. Для обхода дерева по уровням идеальным выбором является **Поиск в ширину (BFS — Breadth-First Search)**.
- **Выбор структуры данных:** Обход в ширину всегда опирается на **Очередь (Queue)**, работающую по принципу FIFO (первым пришел — первым ушел). В Python для этого используется `collections.deque`, так как операции добавления в конец и извлечения из начала работают за `O(1)` (в отличие от обычного списка `list`, где `pop(0)` занимает `O(N)`).
- **Разделение по уровням:** Главный технический нюанс — как понять, где заканчивается один уровень и начинается следующий. Для этого в начале каждой итерации цикла мы фиксируем текущий размер очереди `level_size = len(queue)`. Все элементы, которые сейчас находятся в очереди — это узлы текущего уровня. Мы обрабатываем ровно `level_size` узлов, а их потомков добавляем в конец очереди (они попадут уже на следующий уровень).

### 2. Алгоритм

1. Если корень `root` пуст (`None`), сразу возвращаем пустой список `[]`.
2. Создаем результирующий список `result = []` и очередь `queue = deque([root])`.
3. Пока очередь не пуста:
   - Фиксируем количество узлов на текущем уровне: `level_size = len(queue)`.
   - Создаем временный список `current_level = []` для хранения значений этого уровня.
   - Запускаем цикл ровно `level_size` раз:
     - Извлекаем первый узел из очереди: `node = queue.popleft()`.
     - Добавляем `node.val` в `current_level`.
     - Если у узла есть левый потомок (`node.left`), добавляем его в очередь.
     - Если у узла есть правый потомок (`node.right`), добавляем его в очередь.
   - Добавляем заполненный `current_level` в общий `result`.
4. Вернуть `result`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N)**, где N — общее количество узлов в дереве. Мы посещаем и извлекаем из очереди каждый узел ровно один раз.
- **Сложность по памяти (Space Complexity):** **O(N)**. В худшем случае (для сбалансированного дерева) на самом нижнем уровне будет находиться примерно `N/2` узлов, которые одновременно будут храниться в очереди `queue`.
