# [143. Reorder List](https://leetcode.com/problems/reorder-list/)

## Описание задачи

Дан заголовок односвязного списка. Список может быть представлен как:
`L0 → L1 → … → Ln-1 → Ln`

Необходимо изменить порядок узлов в списке так, чтобы он принял следующий вид:
`L0 → Ln → L1 → Ln-1 → L2 → Ln-2 → …`

Вы не можете изменять значения в узлах списка. Разрешается изменять только сами ссылки (указатели) между узлами.

**Пример 1:**

- **Input:** head = [1,2,3,4]
- **Output:** [1,4,2,3]

**Пример 2:**

- **Input:** head = [1,2,3,4,5]
- **Output:** [1,5,2,4,3]

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Проблема прямого доступа:** Мы не можем легко обращаться к элементам с конца, как в обычном массиве. Если использовать дополнительный массив для хранения всех узлов, мы потратим $O(N)$ памяти, что не оптимально.
- **Декомпозиция задачи:** Если присмотреться к итоговому списку `L0 → Ln → L1 → Ln-1`, можно заметить, что он состоит из двух половин:
  1. Первая половина списка идет в прямом порядке (`L0, L1, L2`).
  2. Вторая половина идет в обратном порядке (`Ln, Ln-1, Ln-2`), и они чередуются друг с другом.
- **Идеальный план (3 фазы):**
  1. **Найти середину списка.** Используем паттерн "Быстрый и медленный указатели" (Заяц и Черепаха). Когда быстрый добежит до конца, медленный окажется ровно на середине.
  2. **Развернуть вторую половину.** Отрезаем вторую половину от первой и применяем к ней классический алгоритм разворота списка in-place (как в задаче 206).
  3. **Слить две половины.** Берем по одному узлу из первой и развернутой второй половины и связываем их "змейкой", перекидывая указатели.

### 2. Алгоритм

1. **Поиск середины:**
   - Инициализировать `slow = head`, `fast = head.next`.
   - Пока `fast` и `fast.next` существуют, сдвигать `slow` на 1 шаг, а `fast` на 2 шага.
2. **Разворот второй половины:**
   - Установить указатель на начало второй половины `second = slow.next`.
   - Разорвать связь между половинами: `slow.next = None`.
   - Развернуть `second` с помощью указателя `prev` (аналогично 206. Reverse Linked List).
3. **Слияние "змейкой":**
   - Теперь у нас есть `first = head` (первая половина) и `second = prev` (развернутая вторая половина).
   - Пока `second` существует:
     - Сохранить ссылки на следующие узлы: `tmp1 = first.next`, `tmp2 = second.next`.
     - Связать `first` с `second` (`first.next = second`).
     - Связать `second` со следующим узлом первой половины (`second.next = tmp1`).
     - Сдвинуть указатели: `first = tmp1`, `second = tmp2`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **$O(N)$**, где N — количество узлов. Поиск середины занимает $O(N/2)$, разворот $O(N/2)$ и слияние $O(N/2)$. В сумме это линейное время.
- **Сложность по памяти (Space Complexity):** **$O(1)$**. Мы только переставляем существующие указатели, дополнительная память для хранения узлов не требуется.
