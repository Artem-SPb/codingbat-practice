# [994. Rotting Oranges](https://leetcode.com/problems/rotting-oranges/)

## Описание задачи

Дана матрица `grid` размером `m x n`, где каждая ячейка может иметь одно из трех значений:

- `0` — пустая ячейка;
- `1` — свежий апельсин;
- `2` — гнилой апельсин.

Каждую минуту любой свежий апельсин, который соседствует (по вертикали или горизонтали) с гнилым апельсином, также становится гнилым.

Необходимо вернуть минимальное количество минут, которое должно пройти, чтобы в сетке не осталось ни одного свежего апельсина. Если это невозможно (какой-то апельсин изолирован), вернуть `-1`.

**Пример 1:**

- **Input:** grid = [[2,1,1],[1,1,0],[0,1,1]]
- **Output:** 4

**Пример 2:**

- **Input:** grid = [[2,1,1],[0,1,1],[1,0,1]]
- **Output:** -1
- **Пояснение:** Апельсин в левом нижнем углу никогда не сгниет, так как он изолирован нулями.

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Выбор алгоритма:** Нам нужно найти _минимальное время_ распространения. Это классический маркер для **Поиска в ширину (BFS)**. Гниль распространяется кругами (уровнями) минута за минутой.
- **Специфика (Multi-source):** В отличие от стандартного BFS, который начинается с одной точки, здесь гниль может начать распространяться сразу из нескольких зараженных апельсинов одновременно.
- **Решение:** Нам нужна очередь (`deque`).
  1. Сначала мы пройдем по всей матрице и положим координаты _всех_ изначально гнилых апельсинов в очередь (это будет наш 0-й уровень, или 0-я минута). Заодно подсчитаем общее количество свежих апельсинов.
  2. Затем запускаем цикл BFS: извлекаем гнилые апельсины текущей минуты, "заражаем" их свежих соседей, уменьшаем счетчик свежих апельсинов и кладем новых зараженных в очередь для следующей минуты.
  3. В конце проверяем: если остались свежие апельсины (счетчик > 0), возвращаем `-1`. Иначе — количество затраченных минут.

### 2. Алгоритм

1. Инициализировать очередь `queue` и переменную `fresh_count = 0`.
2. Пройти по сетке двойным циклом:
   - Если `grid[r][c] == 2`, добавить `(r, c)` в `queue`.
   - Если `grid[r][c] == 1`, увеличить `fresh_count` на 1.
3. Инициализировать `minutes = 0`.
4. Запустить цикл, пока `queue` не пуста и `fresh_count > 0`:
   - Взять текущий размер очереди `level_size = len(queue)` (это все апельсины, гниющие на текущей минуте).
   - Для каждого из `level_size` апельсинов:
     - Извлечь его координаты из очереди.
     - Проверить 4 соседей (вверх, вниз, влево, вправо).
     - Если сосед — свежий апельсин (`1`), сделать его гнилым (`2`), уменьшить `fresh_count` на 1 и добавить его координаты в `queue`.
   - Увеличить `minutes` на 1.
5. Вернуть `minutes`, если `fresh_count == 0`, иначе `-1`.

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(M \* N)**, где M — строки, N — столбцы. В худшем случае мы посетим каждую ячейку матрицы константное количество раз.
- **Сложность по памяти (Space Complexity):** **O(M \* N)**. В худшем случае, если вся матрица заполнена гнилыми апельсинами, очередь будет содержать $M \times N$ элементов.
