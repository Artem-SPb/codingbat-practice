# [207. Course Schedule](https://leetcode.com/problems/course-schedule/)

## Описание задачи

Вам необходимо пройти всего `numCourses` курсов, помеченных от `0` до `numCourses - 1`. Вы получаете массив `prerequisites`, где `prerequisites[i] = [a_i, b_i]` означает, что для того, чтобы взять курс `a_i`, вы **сначала** должны пройти курс `b_i`.

Например, пара `[0, 1]` означает: чтобы пройти курс 0, нужно сначала завершить курс 1.

Верните `true`, если вы можете завершить все курсы. В противном случае верните `false`.

**Пример 1:**

- **Input:** numCourses = 2, prerequisites = [[1,0]]
- **Output:** true
- **Пояснение:** Чтобы взять курс 1, нужно пройти курс 0. Это возможно.

**Пример 2:**

- **Input:** numCourses = 2, prerequisites = [[1,0],[0,1]]
- **Output:** false
- **Пояснение:** Чтобы взять курс 1, нужно пройти курс 0, а чтобы взять курс 0, нужно пройти курс 1. Зацикливание, пройти курсы невозможно.

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Графовая модель:** Эта задача идеально ложится на структуру ориентированного графа. Каждый курс — это вершина. Зависимость (пререквизит) — это направленное ребро от `b_i` к `a_i`.
- **Суть проблемы:** Невозможность пройти все курсы возникает только в одном случае — если в графе есть **цикл** (взаимная зависимость). Значит, задача сводится к поиску цикла в ориентированном графе.
- **Алгоритм обхода:** Для поиска циклов отлично подходит Поиск в глубину (DFS). Нам нужно отслеживать состояния каждой вершины.
- **Три состояния (Цвета вершин):**
  1. `0` (Не посещена / Белая): Мы еще не были в этой вершине.
  2. `1` (В процессе / Серая): Мы зашли в эту вершину и сейчас исследуем её соседей (или соседей её соседей). Если во время обхода мы снова натыкаемся на вершину со статусом `1`, значит, мы нашли цикл!
  3. `2` (Полностью обработана / Черная): Мы и все её соседи успешно проверены, циклов не найдено.

### 2. Алгоритм

1. Построить список смежности `adj`, где ключом будет номер курса, а значением — список курсов, которые зависят от него.
2. Создать массив `states` размером `numCourses`, заполненный `0` (все вершины не посещены).
3. Определить рекурсивную функцию `dfs(node)`:
   - Если `states[node] == 1`, мы нашли цикл (возвращаем `False`).
   - Если `states[node] == 2`, вершина уже проверена, цикла нет (возвращаем `True`).
   - Помечаем вершину как "в процессе": `states[node] = 1`.
   - Запускаем цикл по всем соседям вершины `neighbor` в `adj[node]`:
     - Если `dfs(neighbor)` возвращает `False`, пробрасываем `False` дальше наверх.
   - Помечаем вершину как "полностью обработана": `states[node] = 2`.
   - Возвращаем `True`.
4. Запустить `dfs` для каждого курса от `0` до `numCourses - 1`. (Это необходимо, так как граф может состоять из нескольких несвязных компонент).
5. Если хотя бы один `dfs` вернул `False`, возвращаем `False`. Иначе `True`.

## Анализ сложности

- **Сложность по времени (Time Complexity):** O(V + E), где V — количество курсов (вершин), а E — количество зависимостей (ребер). Мы инициализируем граф и посещаем каждую вершину и ребро не более одного раза.
- **Сложность по памяти (Space Complexity):** O(V + E). Память уходит на хранение списка смежности `adj`, массива состояний `states` и на стек вызовов рекурсии (который в худшем случае может достигнуть глубины V).
