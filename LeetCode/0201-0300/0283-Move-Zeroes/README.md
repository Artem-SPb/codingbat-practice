# [283. Move Zeroes](https://leetcode.com/problems/move-zeroes/)

## Описание задачи

Дан целочисленный массив `nums`. Необходимо написать функцию, которая переместит все нули в конец массива, сохранив относительный порядок ненулевых элементов.

**Ограничение:** Задача должна быть решена **in-place** (на месте), без копирования массива.

**Примеры:**

- **Input:** nums = [0, 1, 0, 3, 12]
- **Output:** [1, 3, 12, 0, 0]

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Подход с дополнительной памятью:** Самый простой способ — создать новый массив, пройтись по исходному, перенести все ненулевые элементы в новый, а затем дописать нули. Но условие требует решения _in-place_, так что этот вариант отпадает.
- **Оптимальный подход (Two Pointers):** Мы можем использовать два указателя для модификации массива на лету.
  - Один указатель (`current` или просто индекс цикла `i`) будет проходить по каждому элементу массива.
  - Второй указатель (`insert_pos`) будет указывать на позицию, куда нужно записать следующий найденный ненулевой элемент.

Всякий раз, когда `current` находит ненулевой элемент, мы меняем местами элементы под указателями `current` и `insert_pos`, а затем сдвигаем `insert_pos` на один шаг вперед. Таким образом, все ненулевые элементы постепенно "сдвигаются" влево, сохраняя свой порядок, а нули автоматически "выталкиваются" вправо.

### 2. Алгоритм

1. Инициализировать указатель `insert_pos = 0`. Он будет отслеживать позицию для вставки следующего ненулевого числа.
2. Запустить цикл `for`, где индекс `i` (наш `current` указатель) проходит от `0` до конца массива.
3. Внутри цикла проверяем: если `nums[i] != 0`:
   - Меняем местами элементы `nums[insert_pos]` и `nums[i]`.
   - Увеличиваем `insert_pos` на 1.
4. По завершении цикла все ненулевые элементы окажутся в начале массива в правильном порядке, а оставшаяся часть будет заполнена нулями.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(n)**, так как мы проходим по массиву `nums` ровно один раз.
- **Сложность по памяти (Space Complexity):** **O(1)**, так как мы модифицируем массив на месте и используем только одну дополнительную переменную (`insert_pos`).

---

## Ключевые выводы / Паттерн

- **Two Pointers (Два указателя) одного направления (Slow & Fast Pointers):** Этот паттерн отлично подходит для фильтрации, удаления дубликатов или перестановки элементов массива in-place. Медленный указатель следит за позицией для записи, а быстрый — ищет подходящие элементы.
