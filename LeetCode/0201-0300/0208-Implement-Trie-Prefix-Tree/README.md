# [208. Implement Trie (Prefix Tree)](https://leetcode.com/problems/implement-trie-prefix-tree/)

## Описание задачи

Префиксное дерево (Trie) — это древовидная структура данных, используемая для эффективного хранения и извлечения ключей в наборе строк.

Необходимо реализовать класс `Trie`:

- `Trie()` инициализирует объект префиксного дерева.
- `void insert(String word)` вставляет строку `word` в дерево.
- `boolean search(String word)` возвращает `true`, если строка `word` есть в дереве (т.е. была вставлена ранее), и `false` в противном случае.
- `boolean startsWith(String prefix)` возвращает `true`, если в дереве есть ранее вставленная строка, которая начинается с префикса `prefix`, и `false` в противном случае.

**Пример:**

> Trie trie = new Trie();
> trie.insert("apple");
> trie.search("apple"); // возвращает True
> trie.search("app"); // возвращает False
> trie.startsWith("app"); // возвращает True
> trie.insert("app");
> trie.search("app"); // возвращает True

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Структура узла:** В отличие от бинарного дерева (где есть только `left` и `right`), узел префиксного дерева может иметь до 26 потомков (по количеству букв английского алфавита). Каждый узел должен хранить ссылки на своих детей. В Python для этого удобнее всего использовать словарь (`dict`). Также каждому узлу нужен флаг `is_end_of_word`, чтобы отличать полное слово от просто префикса.
- **Вставка (Insert):** Мы начинаем с корня. Для каждой буквы в слове проверяем, есть ли она среди детей текущего узла. Если нет — создаем новый узел. Переходим к узлу-ребенку. По окончании слова помечаем последний узел как `is_end_of_word = True`.
- **Поиск (Search):** Идем от корня по буквам слова. Если на каком-то шаге нужной буквы нет в детях — возвращаем `False`. Если дошли до конца слова, проверяем флаг `is_end_of_word`.
- **Поиск префикса (StartsWith):** Логика абсолютно идентична `search`, но в конце нам не важно, стоит ли флаг `is_end_of_word`. Достаточно того, что мы смогли пройти по всему префиксу, не обрываясь.

### 2. Алгоритм

1. Создать вспомогательный класс `TrieNode` с атрибутами `children` (словарь) и `is_end_of_word` (булево значение, по умолчанию `False`).
2. В `__init__` класса `Trie` создать корневой узел `self.root = TrieNode()`. Корневой узел не хранит букву, он является стартовой точкой.
3. В методе `insert(word)` проходить по символам. Для каждого символа добавлять новый `TrieNode` в `children` текущего узла, если его там еще нет, и сдвигать указатель `curr` вниз. В конце цикла установить `curr.is_end_of_word = True`.
4. В методе `search(word)` спускаться по узлам. Вернуть `False` при первом несовпадении. В конце вернуть значение `curr.is_end_of_word`.
5. В методе `startsWith(prefix)` делать то же самое, но в конце всегда возвращать `True`.

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(M)** для всех операций (`insert`, `search`, `startsWith`), где M — длина слова. Нам нужно выполнить ровно M шагов, чтобы пройти по дереву.
- **Сложность по памяти (Space Complexity):** **O(N \* M)** в худшем случае для метода `insert` (где N — количество вставленных слов, а M — их максимальная длина), если все слова имеют разные префиксы. Операции `search` и `startsWith` требуют **O(1)** дополнительной памяти.
