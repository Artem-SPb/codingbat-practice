# [230. Kth Smallest Element in a BST](https://leetcode.com/problems/kth-smallest-element-in-a-bst/)

---

### 🇷🇺 Русская версия

## Описание задачи

Дан корневой узел `root` Бинарного Дерева Поиска (BST) и целое число `k`. Верните `k`-й по величине (1-индексированный) элемент среди всех значений узлов дерева.

**Пример:**
**Input:** root = [3, 1, 4, null, 2], k = 1
**Output:** 1

---

## Мой подход к решению

### 1. Размышления (Thought Process)

Ключевая фраза — "k-й наименьший элемент". В контексте BST это сразу должно вызывать ассоциацию с **отсортированным порядком**.

Главное свойство BST заключается в том, что **in-order обход** (рекурсивный обход в порядке: Левое поддерево -> Корень -> Правое поддерево) посещает все узлы в порядке возрастания их значений.

- **Простой, но неоптимальный подход:**

  1.  Написать рекурсивную функцию для in-order обхода.
  2.  Собрать значения всех узлов в список.
  3.  Вернуть элемент списка по индексу `k-1`.

  - **Недостаток:** Этот метод всегда обходит всё дерево и требует O(n) дополнительной памяти для списка, даже если `k=1`.

- **Оптимальный итеративный подход:**
  Можно ли остановиться, как только мы найдем k-й элемент? Да. Для этого идеально подходит **итеративный in-order обход с использованием стека**.
  Идея в том, чтобы симулировать рекурсивный обход:
  1.  Мы идем по левым потомкам до самого конца, складывая все узлы по пути в стек. Самый верхний узел в стеке после этого — самый маленький элемент в дереве.
  2.  Затем мы достаем узел из стека — это наш "посещенный" узел. Мы уменьшаем счетчик `k`. Если `k` стало равно нулю, мы нашли то, что искали.
  3.  После "посещения" узла мы должны обработать его правое поддерево. Мы переходим к правому потомку и повторяем шаг 1 для него.

Этот подход позволяет нам остановиться ровно в тот момент, когда k-й элемент найден, что в среднем эффективнее.

### 2. Алгоритм (Итеративный)

1.  Инициализировать пустой `stack` и `current = root`.
2.  Запустить основной цикл `while`, который работает, пока `current` не `None` или `stack` не пуст.
3.  Внутри цикла:
    a. Вложенный цикл `while current is not None`: добавлять `current` в `stack` и переходить влево (`current = current.left`).
    b. Когда левый путь исчерпан, достать узел из стека: `current = stack.pop()`.
    c. Уменьшить `k` на 1. Если `k` стало равно 0, вернуть `current.val`.
    d. Перейти к обработке правого поддерева: `current = current.right`.

## Код решения

```python
from typing import Optional

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        stack = []
        current = root

        while current or stack:
            while current:
                stack.append(current)
                current = current.left

            current = stack.pop()

            k -= 1
            if k == 0:
                return current.val

            current = current.right
```
