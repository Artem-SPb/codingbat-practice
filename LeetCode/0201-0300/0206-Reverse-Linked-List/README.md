# [206. Reverse Linked List](https://leetcode.com/problems/reverse-linked-list/)

## Описание задачи

Дан заголовок (`head`) односвязного списка. Необходимо развернуть список в обратном порядке и вернуть его новый заголовок.

**Пример 1:**

- **Input:** head = [1,2,3,4,5]
- **Output:** [5,4,3,2,1]

**Пример 2:**

- **Input:** head = [1,2]
- **Output:** [2,1]

**Пример 3:**

- **Input:** head = []
- **Output:** []

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Специфика односвязного списка:** Каждый узел (`ListNode`) хранит только своё значение (`val`) и ссылку на следующий узел (`next`). Шагнуть назад по умолчанию нельзя.
- **Идея разворота "на месте" (In-place Iterative):** Чтобы развернуть список, нам нужно для каждого узла поменять направление его ссылки `next` так, чтобы она указывала на _предыдущий_ узел.
- **Проблема разрыва связи:** Если мы просто сделаем `curr.next = prev`, мы мгновенно потеряем ссылку на остаток списка (то, что раньше лежало в `curr.next`).
- **Решение (Три указателя):** Нам понадобятся три переменные:
  1. `prev` — предыдущий узел (изначально `None`, так как новый хвост списка должен указывать на `None`).
  2. `curr` — текущий обрабатываемый узел (изначально `head`).
  3. `next_temp` — временная переменная для сохранения ссылки на следующий узел перед перезаписью.
- **Шаг итерации:** Запоминаем `next_temp = curr.next`, разворачиваем ссылку назад `curr.next = prev`, сдвигаем указатели вперед: `prev = curr`, `curr = next_temp`.
- Когда `curr` станет равен `None` (мы вышли за пределы списка), указатель `prev` окажется строго на бывшем хвосте, который теперь стал новым заголовком (`head`).

### 2. Алгоритм

1. Инициализировать указатели: `prev = None`, `curr = head`.
2. Пока `curr` не равен `None`:
   - Сохранить ссылку на следующий узел: `next_temp = curr.next`.
   - Развернуть ссылку текущего узла назад: `curr.next = prev`.
   - Сдвинуть `prev` на место `curr`: `prev = curr`.
   - Сдвинуть `curr` на сохраненное место: `curr = next_temp`.
3. Вернуть `prev`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N)**, где N — количество узлов в списке. Мы проходим по каждому узлу ровно один раз.
- **Сложность по памяти (Space Complexity):** **O(1)**. Разворот происходит in-place путем переопределения существующих ссылок, дополнительная память под новые узлы не выделяется.
