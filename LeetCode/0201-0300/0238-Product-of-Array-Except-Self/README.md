# [238. Product of Array Except Self](https://leetcode.com/problems/product-of-array-except-self/)

## Описание задачи

Вам дан целочисленный массив `nums`. Необходимо вернуть массив `answer`, в котором `answer[i]` равен произведению всех элементов массива `nums`, за исключением `nums[i]`.

Гарантируется, что произведение префиксов и суффиксов для любого префикса или суффикса массива помещается в 32-битное целое число.

**Строгое ограничение:** Вы должны решить задачу за время **O(N)** и **без использования операции деления**.

**Пример 1:**

- **Input:** nums = [1,2,3,4]
- **Output:** [24,12,8,6]

**Пример 2:**

- **Input:** nums = [-1,1,0,-3,3]
- **Output:** [0,0,9,0,0]

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Проблема деления:** Если бы деление было разрешено, мы бы просто перемножили все числа массива, а затем для каждого `i` разделили бы общую сумму на `nums[i]`. Но деление запрещено условием, плюс проблема с нулями (деление на ноль).
- **Что такое "произведение всех, кроме самого себя"?** Для любого элемента массива по индексу `i`, его результат — это произведение всех элементов, стоящих _слева_ от него, умноженное на произведение всех элементов, стоящих _справа_ от него.
- **Паттерн Prefix & Postfix (Префиксы и Постфиксы):**
  1. Мы можем пройти по массиву слева направо и записать для каждого элемента произведение всего, что было до него (префикс).
  2. Затем пройти справа налево и домножить уже записанный результат на произведение всего, что стоит после него (постфикс).
- **Оптимизация памяти:** Чтобы не создавать два отдельных массива под префиксы и постфиксы (что даст память O(N)), мы можем сразу записывать результаты в итоговый массив `answer`. Переменные для накопления текущего префикса и постфикса мы будем хранить "на лету" (в памяти O(1)). По условиям LeetCode, память под массив ответа не учитывается в оценке Space Complexity.

### 2. Алгоритм

1. Создать массив `answer` такого же размера, как и `nums`, заполненный единицами.
2. Инициализировать переменную `prefix = 1`.
3. Пройти циклом слева направо (`for i in range(len(nums))`):
   - Записать текущий `prefix` в `answer[i]`.
   - Домножить `prefix` на `nums[i]`, чтобы он был готов для следующего шага.
4. Инициализировать переменную `postfix = 1`.
5. Пройти циклом справа налево (`for i in range(len(nums) - 1, -1, -1)`):
   - Домножить `answer[i]` на текущий `postfix`.
   - Домножить `postfix` на `nums[i]`, чтобы он был готов для следующего шага влево.
6. Вернуть `answer`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N)**, где N — длина массива. Мы делаем ровно два независимых прохода по массиву.
- **Сложность по памяти (Space Complexity):** **O(1)**. Мы используем только две целочисленные переменные `prefix` и `postfix`. Возвращаемый массив `answer` не учитывается в оценке дополнительной памяти.
