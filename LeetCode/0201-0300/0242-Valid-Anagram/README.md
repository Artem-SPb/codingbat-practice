# [242. Valid Anagram](https://leetcode.com/problems/valid-anagram/)

## Описание задачи

Даны две строки `s` и `t`. Необходимо вернуть `true`, если строка `t` является анаграммой строки `s`, и `false` в противном случае.

**Анаграмма** — это слово или фраза, образованная путем перестановки букв другого слова или фразы, как правило, с использованием всех исходных букв ровно один раз.

**Пример 1:**
**Input:** s = "anagram", t = "nagaram"
**Output:** true

**Пример 2:**
**Input:** s = "rat", t = "car"
**Output:** false

---

## Мой подход к решению

### 1. Размышления (Thought Process)

Ключевое свойство анаграмм: они состоят из **одного и того же набора символов** с **одинаковой частотой** каждого символа.

- **Подход 1 (Сортировка):** Если отсортировать обе строки, и они станут равны, значит, это анаграммы. Например, `sorted("anagram")` и `sorted("nagaram")` оба превратятся в `['a', 'a', 'a', 'g', 'm', 'n', 'r']`. Это простое и элегантное решение. Однако его временная сложность определяется сортировкой — **O(n log n)**.

- **Оптимальный подход (Хэш-таблица / Счетчик частот):** Можно ли быстрее? Да, за линейное время **O(n)**. Для этого нужно посчитать, сколько раз каждый символ встречается в первой строке, а затем проверить, соответствует ли вторая строка этим подсчетам. Хэш-таблица (словарь в Python) идеально подходит для этой роли: `{символ: количество}`.

Также, есть очевидная **оптимизация**: если длины строк не равны, они точно не могут быть анаграммами. Эту проверку нужно сделать в самом начале, чтобы избежать лишней работы.

Я выберу второй подход с использованием хэш-таблицы как более эффективный по времени.

### 2. Алгоритм

1.  Проверить, равна ли длина `s` и `t`. Если нет, немедленно вернуть `False`.
2.  Создать хэш-таблицу (словарь) `s_counts` для хранения частот символов.
3.  Пройти по первой строке `s` и заполнить `s_counts`. Для каждого символа увеличивать его счетчик.
4.  Пройти по второй строке `t`. Для каждого символа `char`:
    - Проверить, есть ли он в `s_counts` и больше ли его счетчик нуля. Если нет — строки не совпадают, вернуть `False`.
    - Если проверка пройдена, уменьшить счетчик для этого символа: `s_counts[char] -= 1`.
5.  Если цикл по строке `t` успешно завершился, это означает, что все символы и их частоты совпали. Вернуть `True`.

---

## Код решения

```python
class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        if len(s) != len(t):
            return False

        s_counts = {}

        for char in s:
            s_counts[char] = s_counts.get(char, 0) + 1

        for char in t:
            if char not in s_counts or s_counts[char] == 0:
                return False
            s_counts[char] -= 1

        return True
```

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(n)**, где `n` — длина строки `s` (или `t`). Мы делаем два отдельных прохода по строкам, что в сумме дает O(n + n) = O(n).
- **Сложность по памяти (Space Complexity):** **O(k)**, где `k` — количество уникальных символов. Поскольку в условии задачи сказано, что строки состоят из строчных букв английского алфавита, `k` не может быть больше 26. Таким образом, сложность можно считать константной, то есть **O(1)**.

## Ключевые выводы / Паттерн

- **Подсчет частот (Frequency Counting):** Использование хэш-таблицы для подсчета количества вхождений элементов (символов, чисел и т.д.) — это один из самых распространенных и мощных паттернов. Он позволяет решать множество задач за линейное время.
- **Ранний выход (Early Exit):** Простые проверки в начале функции (например, на равенство длин), которые позволяют сразу отсечь невозможные случаи, — это признак хорошего, чистого кода.
