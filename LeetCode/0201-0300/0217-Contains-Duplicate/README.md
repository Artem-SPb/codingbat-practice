# [217. Contains Duplicate](https://leetcode.com/problems/contains-duplicate/)

## Описание задачи

Дан массив целых чисел `nums`. Необходимо вернуть `true`, если любое значение появляется в массиве хотя бы дважды, и `false`, если каждый элемент уникален.

**Пример 1:**
**Input:** nums = [1, 2, 3, 1]
**Output:** true

**Пример 2:**
**Input:** nums = [1, 2, 3, 4]
**Output:** false

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Подход "в лоб" (Brute-force):** Сравнить каждый элемент с каждым. Сложность **O(n²)**. Неэффективно.
- **Подход с сортировкой:** Отсортировать массив за **O(n log n)**. Затем пройти по нему один раз и сравнить соседние элементы. Если `nums[i] == nums[i+1]`, значит, есть дубликат. Это хороший подход, если нельзя использовать дополнительную память.
- **Оптимальный подход (Хэш-сет):** Задача сводится к вопросу: "Встречал ли я это число раньше?". Этот вопрос — прямой сигнал к использованию **хэш-сета** (`set` в Python). Сет хранит только уникальные элементы и позволяет проверять наличие элемента в нем за **O(1)** в среднем.

Идея проста: мы итерируемся по массиву и складываем числа в сет, который будет играть роль нашей "памяти". Перед тем как добавить очередное число, мы проверяем, нет ли его уже в сете. Если есть — мы нашли дубликат.

### 2. Алгоритм

1.  Создать пустой хэш-сет `hashset`.
2.  Начать итерацию по массиву `nums`.
3.  Для каждого числа `num`:
    - Проверить, содержится ли `num` в `hashset`.
    - Если да, немедленно вернуть `True`.
    - Если нет, добавить `num` в `hashset`.
4.  Если цикл успешно завершился (то есть `return True` не сработал), значит, дубликатов нет. Вернуть `False`.

---

## Код решения

```python
from typing import List

class Solution:
    def containsDuplicate(self, nums: List[int]) -> bool:
        hashset = set()

        for num in nums:
            if num in hashset:
                return True
            hashset.add(num)

        return False
```

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(n)**, так как мы проходим по массиву `nums` ровно один раз. Операции `in` и `add` для сета в среднем занимают O(1).
- **Сложность по памяти (Space Complexity):** **O(n)**, так как в худшем случае (если все элементы уникальны) нам придется сохранить в `hashset` все `n` элементов.

---

## Ключевые выводы / Паттерн

- **Отслеживание виденного (Tracking Seen Elements):** Хэш-сет — это идеальный инструмент для отслеживания элементов, которые вы уже встречали в коллекции. Он обеспечивает самую быструю проверку на уникальность. Этот паттерн применяется в задачах на поиск дубликатов, уникальных элементов, пересечений и объединений множеств.
