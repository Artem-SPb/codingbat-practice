# [424. Longest Repeating Character Replacement](https://leetcode.com/problems/longest-repeating-character-replacement/)

## Описание задачи

Дана строка `s`, состоящая только из заглавных английских букв, и целое число `k`. Вы можете заменить любой символ в строке на любую другую заглавную английскую букву не более `k` раз.

Необходимо найти длину самой длинной подстроки, содержащей одинаковые буквы, которую можно получить после выполнения не более чем `k` замен.

**Пример 1:**

- **Input:** s = "ABAB", k = 2
- **Output:** 4
- **Пояснение:** Заменяем обе буквы 'A' на 'B' или обе 'B' на 'A', получаем "BBBB" или "AAAA". Длина равна 4.

**Пример 2:**

- **Input:** s = "AABABBA", k = 1
- **Output:** 4
- **Пояснение:** Заменяем одну букву 'A' в середине на 'B', получаем "AABBBBA". Самая длинная подстрока с одинаковыми буквами — "BBBB" (длина 4).

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Проблема:** Нам нужно найти самую длинную валидную подстроку. Подстрока считается валидной, если количество "лишних" символов в ней (тех, которые нужно заменить) не превышает `k`.
- **Как посчитать "лишние" символы?** Длина текущей подстроки минус количество самых частых символов в этой подстроке. Например, в "AABA" длина 4, самый частый символ 'A' встречается 3 раза. Значит, лишних символов: 4 - 3 = 1. Если `k >= 1`, эта подстрока валидна.
- **Оптимальный подход (Sliding Window):** Мы будем расширять наше "окно" (двигать правый указатель), добавляя символы в хеш-таблицу для подсчета их частоты. Как только окно становится невалидным (лишних символов > `k`), мы начинаем сужать его слева (двигать левый указатель), пока оно снова не станет валидным.

### 2. Алгоритм

1. Инициализировать указатель `left = 0`, словарь `counts` для частот символов и `max_freq = 0` (частота самого популярного символа в текущем окне).
2. Запустить цикл с правым указателем `right` от 0 до конца строки:
   - Добавить текущий символ `s[right]` в `counts` и обновить `max_freq`.
   - Вычислить текущую длину окна: `window_len = right - left + 1`.
   - Если `window_len - max_freq > k`, значит окно невалидно. Нам нужно сдвинуть `left` вправо. При этом мы уменьшаем счетчик символа `s[left]` в `counts` и делаем `left += 1`.
   - Максимальная длина валидного окна на протяжении всего алгоритма и будет нашим ответом.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N)**. Оба указателя (`left` и `right`) проходят по строке только один раз. Внутри цикла мы выполняем операции за константное время.
- **Сложность по памяти (Space Complexity):** **O(1)**. Наш словарь `counts` будет хранить максимум 26 ключей (буквы английского алфавита), что является константой и не зависит от длины строки `s`.
