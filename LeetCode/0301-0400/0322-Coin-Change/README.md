# [322. Coin Change](https://leetcode.com/problems/coin-change/)

## Описание задачи

Вам дан целочисленный массив `coins`, представляющий монеты различных номиналов, и целое число `amount`, представляющее общую сумму денег.

Необходимо вернуть **минимальное количество монет**, из которых можно собрать эту сумму. Если из заданных номиналов монет собрать сумму невозможно, верните `-1`.

Вы можете считать, что у вас есть бесконечное количество монет каждого номинала.

**Пример 1:**

- **Input:** coins = [1,2,5], amount = 11
- **Output:** 3
- **Пояснение:** 11 = 5 + 5 + 1 (3 монеты).

**Пример 2:**

- **Input:** coins = [2], amount = 3
- **Output:** -1

**Пример 3:**

- **Input:** coins = [1], amount = 0
- **Output:** 0

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Жадный алгоритм (Greedy) не сработает:** Интуитивно хочется всегда брать самую большую монету (например, для суммы 11 брать пятерки). Но если номиналы `[3, 4, 5]` и нужна сумма `7`, жадный алгоритм возьмет `5` и остановится, так как `2` не собрать. А правильный ответ `3 + 4` (2 монеты).
- **Динамическое программирование (Bottom-Up):** Нам нужно разбить большую задачу (собрать `amount`) на мелкие подзадачи (как собрать сумму `0`, `1`, `2` ... `amount` оптимальным образом).
- **Состояние (State):** Заведем массив `dp`, где `dp[i]` будет хранить минимальное количество монет для суммы `i`.
- **Базовый случай:** `dp[0] = 0` (чтобы собрать сумму 0, нужно 0 монет). Все остальные значения инициализируем бесконечностью `inf`, так как мы ищем минимум.
- **Формула перехода:** Чтобы собрать сумму `i`, мы можем попробовать прибавить каждую из доступных монет `c`. Тогда `dp[i] = min(dp[i], dp[i - c] + 1)`.
- _Пример:_ Чтобы собрать сумму 11 имея монету 5, мы смотрим, за сколько монет мы собрали сумму 6 (т.е. `11 - 5`), и прибавляем к этому 1 (саму монету 5).

### 2. Алгоритм

1. Создать массив `dp` размером `amount + 1` и заполнить его значением `infinity` (бесконечность).
2. Задать базовый случай: `dp[0] = 0`.
3. Запустить цикл от `i = 1` до `amount`:
   - Внутри запустить цикл по всем номиналам `coin` из `coins`.
   - Если текущая монета помещается в сумму (`i - coin >= 0`), обновляем минимум: `dp[i] = min(dp[i], dp[i - coin] + 1)`.
4. Если `dp[amount]` осталось равно `infinity`, значит сумму собрать невозможно — возвращаем `-1`. В противном случае возвращаем `dp[amount]`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(S \* N)**, где S — целевая сумма (`amount`), а N — количество номиналов монет (`len(coins)`). Мы используем вложенный цикл: внешний до `amount`, внутренний по массиву монет.
- **Сложность по памяти (Space Complexity):** **O(S)**, так как мы выделяем массив `dp` размером `amount + 1` для мемоизации подзадач.
