# [15. 3Sum](https://leetcode.com/problems/3sum/)

## Описание задачи

Дан целочисленный массив `nums`. Необходимо вернуть все уникальные триплеты `[nums[i], nums[j], nums[k]]` такие, что:

- `i != j`, `i != k`, и `j != k` (индексы не совпадают).
- `nums[i] + nums[j] + nums[k] == 0`.

Обратите внимание, что в ответе **не должно быть дублирующихся триплетов** (например, `[-1, 0, 1]` и `[0, 1, -1]` считаются одним и тем же триплетом).

**Пример 1:**

- **Input:** nums = [-1,0,1,2,-1,-4]
- **Output:** [[-1,-1,2],[-1,0,1]]
- **Пояснение:**
  nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0.
  nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0.
  nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0.
  Уникальные триплеты: [-1,0,1] и [-1,-1,2].

**Пример 2:**

- **Input:** nums = [0,1,1]
- **Output:** []
- **Пояснение:** Невозможно найти три числа, сумма которых равна 0.

**Пример 3:**

- **Input:** nums = [0,0,0]
- **Output:** [[0,0,0]]

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Свести к известному:** Задача найти три числа, дающих в сумме 0 (`a + b + c = 0`), эквивалентна задаче найти два числа, дающих в сумме `-a` (`b + c = -a`). Это значит, что мы можем зафиксировать одно число, а для оставшейся части массива запустить алгоритм `Two Sum II`!
- **Необходимость сортировки:** Чтобы алгоритм двух указателей из `Two Sum II` сработал, массив должен быть отсортирован. Кроме того, сортировка поможет нам легко избегать дубликатов.
- **Борьба с дубликатами:** Это самая коварная часть задачи.
  1. Если текущее фиксируемое число `a` совпадает с предыдущим, мы его пропускаем (иначе получим те же самые комбинации).
  2. Внутри поиска двух указателей, если мы нашли подходящую пару, нам нужно сдвинуть левый указатель так, чтобы пропустить все одинаковые числа (чтобы не собрать тот же самый триплет снова).

### 2. Алгоритм

1. Отсортировать массив `nums` по возрастанию.
2. Инициализировать результирующий список `res = []`.
3. Запустить цикл по массиву: `for i, a in enumerate(nums):`
   - **Оптимизация:** Если `a > 0`, можно сразу прервать цикл (`break`), так как сумма трех положительных чисел никогда не даст 0.
   - **Пропуск дубликатов `a`:** Если `i > 0` и `a == nums[i - 1]`, переходим к следующей итерации (`continue`).
   - Инициализировать два указателя: `left = i + 1` и `right = len(nums) - 1`.
   - Пока `left < right`:
     - Вычислить текущую сумму `three_sum = a + nums[left] + nums[right]`.
     - Если `three_sum > 0`: сдвинуть `right` влево (`right -= 1`).
     - Если `three_sum < 0`: сдвинуть `left` вправо (`left += 1`).
     - Если `three_sum == 0`:
       - Добавить `[a, nums[left], nums[right]]` в `res`.
       - Сдвинуть `left` вправо (`left += 1`).
       - **Пропуск дубликатов `left`:** Пока `nums[left] == nums[left - 1]` и `left < right`, продолжать сдвигать `left` вправо. (Сдвигать `right` не обязательно, так как при фиксированном `a` и новом `left`, старый `right` уже не даст сумму 0).
4. Вернуть `res`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N²)**. Сортировка занимает `O(N log N)`. Внешний цикл выполняется `N` раз, а внутри него два указателя проходят оставшуюся часть массива за `O(N)`. Итоговая сложность `O(N log N + N²)`, что упрощается до `O(N²)`.
- **Сложность по памяти (Space Complexity):** **O(1)** или **O(N)**. Зависит от встроенного алгоритма сортировки (в Python Timsort требует `O(N)` дополнительной памяти). Исключая память под результирующий массив, дополнительных структур мы не создаем.
