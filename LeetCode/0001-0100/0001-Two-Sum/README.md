# [1. Two Sum](https://leetcode.com/problems/two-sum/)

## Описание задачи

Дан массив целых чисел `nums` и целое число `target`. Необходимо вернуть **индексы** двух чисел из этого массива, которые в сумме дают `target`.

Предполагается, что для каждого набора входных данных существует ровно одно решение, и нельзя использовать один и тот же элемент дважды.

**Пример:**
**Input:** nums = [2, 7, 11, 15], target = 9
**Output:** [0, 1]
**Пояснение:** Потому что nums[0] + nums[1] == 9.

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Подход "в лоб" (Brute-force):** Первое, что приходит на ум, — это использовать вложенные циклы. Внешний цикл берет элемент `i`, а внутренний — перебирает все последующие элементы `j` и проверяет, равна ли их сумма `target`. Этот подход имеет временную сложность **O(n²)**, что слишком медленно для больших массивов.

- **Оптимальный подход (Хэш-таблица):** Чтобы улучшить время, нужно избавиться от внутреннего цикла. Проблема в том, что поиск второго слагаемого (`complement`) занимает O(n) времени. Как можно ускорить этот поиск до O(1)? С помощью хэш-таблицы (в Python — `dict`).

Идея состоит в том, чтобы пройти по массиву один раз. Для каждого элемента `num` мы вычисляем, какое число `complement` нам нужно (`target - num`). Затем мы мгновенно проверяем, встречали ли мы `complement` ранее, посмотрев в нашу хэш-таблицу. Если да — задача решена. Если нет — мы добавляем текущий элемент `num` и его индекс в таблицу, чтобы он мог стать `complement` для будущих элементов.

### 2. Алгоритм

1.  Создать пустой словарь `num_map` для хранения пар `{число: индекс}`.
2.  Начать итерацию по массиву `nums`, получая на каждом шаге индекс `i` и значение `num`.
3.  Вычислить необходимое дополнение: `complement = target - num`.
4.  Проверить, есть ли `complement` в ключах словаря `num_map`.
5.  Если `complement` найден, это означает, что мы нашли пару. Вернуть `[индекс_комплемента, текущий_индекс]`, то есть `[num_map[complement], i]`.
6.  Если `complement` не найден, добавить текущую пару `{num: i}` в словарь.

---

## Код решения

```python
from typing import List

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        num_map = {}  # {value: index}
        for i, num in enumerate(nums):
            complement = target - num
            if complement in num_map:
                return [num_map[complement], i]
            num_map[num] = i
```

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(n)**, так как мы проходим по массиву `nums` ровно один раз. Операции поиска и вставки в хэш-таблицу в среднем занимают O(1).
- **Сложность по памяти (Space Complexity):** **O(n)**, так как в худшем случае (например, если искомая пара — два последних элемента) нам придется сохранить в `num_map` все `n` элементов.

---

## Ключевые выводы / Паттерн

- **Поиск дополнения (Complementary Search):** Если для текущего элемента `x` нужно быстро найти другой элемент `y` по какому-то правилу (например, `y = target - x`), хэш-таблица — идеальный инструмент. Этот паттерн является одним из самых фундаментальных для решения задач на LeetCode.
