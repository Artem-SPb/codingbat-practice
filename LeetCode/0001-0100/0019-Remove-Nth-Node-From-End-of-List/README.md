# [19. Remove Nth Node From End of List](https://leetcode.com/problems/remove-nth-node-from-end-of-list/)

## Описание задачи

Дан заголовок (`head`) односвязного списка. Необходимо удалить `n`-й узел с конца списка и вернуть его заголовок.

**Пример 1:**

- **Input:** head = [1,2,3,4,5], n = 2
- **Output:** [1,2,3,5]
- **Пояснение:** Удален 2-й узел с конца (число 4).

**Пример 2:**

- **Input:** head = [1], n = 1
- **Output:** []

**Пример 3:**

- **Input:** head = [1,2], n = 1
- **Output:** [1]

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Подход "в два прохода":** Самое простое решение — сначала пройти по всему списку, чтобы узнать его длину `L`. Затем вычислить индекс удаляемого узла с начала (`L - n`) и пройти по списку еще раз, чтобы перекинуть ссылку. Это работает за O(N) времени, но требует двух проходов.
- **Оптимальный подход (Fast & Slow Pointers):** Можно ли сделать это за один проход? Да, если использовать два указателя.
- **Идея:** Отправим `fast` указатель на `n` шагов вперед. Теперь расстояние между `fast` и `slow` указателями ровно `n`. Если мы начнем двигать их одновременно по одному шагу, то в тот момент, когда `fast` достигнет самого конца списка, `slow` окажется ровно за `n` шагов до конца!
- **Крайний случай (Удаление головы):** Что если нам нужно удалить самый первый узел (например, `n` равно длине списка)? Чтобы избежать сложных проверок, мы используем паттерн **Dummy Node (фиктивный узел)**. Мы создаем узел-пустышку, который указывает на `head`, и ставим `slow` и `fast` изначально на него. Это гарантирует, что `slow` всегда остановится _прямо перед_ удаляемым узлом, даже если удалять нужно голову.

### 2. Алгоритм

1. Создать фиктивный узел `dummy`, где `dummy.next = head`.
2. Инициализировать два указателя: `slow = dummy` и `fast = dummy`.
3. Сдвинуть `fast` на `n` шагов вперед (циклом `for`).
4. Пока `fast.next` не равен `None` (пока `fast` не станет последним узлом списка):
   - Сдвинуть оба указателя на один шаг: `slow = slow.next`, `fast = fast.next`.
5. Теперь `slow` находится прямо перед узлом, который нужно удалить. Перекидываем ссылку, минуя удаляемый узел: `slow.next = slow.next.next`.
6. Возвращаем `dummy.next` (это безопасный указатель на новую голову списка).

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N)**, где N — количество узлов в списке. Мы совершаем ровно один проход по списку.
- **Сложность по памяти (Space Complexity):** **O(1)**, так как мы используем только несколько дополнительных указателей (`dummy`, `slow`, `fast`).
