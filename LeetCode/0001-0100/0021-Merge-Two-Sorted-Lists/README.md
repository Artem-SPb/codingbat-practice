# [21. Merge Two Sorted Lists](https://leetcode.com/problems/merge-two-sorted-lists/)

## Описание задачи

Вам даны заголовки двух отсортированных связных списков `list1` и `list2`.

Необходимо объединить эти два списка в один отсортированный список. Новый список должен быть составлен путем связывания узлов из первых двух списков.

Верните заголовок объединенного связного списка.

**Пример 1:**

- **Input:** list1 = [1,2,4], list2 = [1,3,4]
- **Output:** [1,1,2,3,4,4]

**Пример 2:**

- **Input:** list1 = [], list2 = []
- **Output:** []

**Пример 3:**

- **Input:** list1 = [], list2 = [0]
- **Output:** [0]

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Проблема старта:** Когда мы создаем новый список, нам нужно вернуть его заголовок (`head`). Но в процессе слияния мы будем постоянно двигаться вперед, добавляя новые узлы. Как не потерять начало?
- **Паттерн Dummy Node (Фиктивный узел):** Мы создаем пустой узел-заглушку `dummy = ListNode()`. Он служит безопасной стартовой точкой. Мы заводим указатель `tail`, который изначально указывает на `dummy`, и используем `tail` для прикрепления новых узлов. В конце мы просто вернем `dummy.next` — это и будет реальное начало нашего слитого списка. Этот трюк избавляет нас от кучи проверок `if not head` внутри цикла.
- **Слияние (Два указателя):** Мы сравниваем значения текущих узлов `list1` и `list2`. Тот, чье значение меньше, прикрепляется к `tail.next`. Затем мы сдвигаем указатель этого списка вперед, а также сдвигаем сам `tail`.
- **Хвосты списков:** Поскольку списки могут быть разной длины, цикл прервется, когда один из них закончится. Оставшийся кусок второго списка уже отсортирован, поэтому мы можем просто прикрепить его целиком за одну операцию присваивания.

### 2. Алгоритм

1. Создать фиктивный узел `dummy = ListNode()`.
2. Создать указатель `tail = dummy`.
3. Пока `list1` и `list2` не равны `None`:
   - Если `list1.val < list2.val`:
     - Прикрепить `list1` к `tail.next`.
     - Сдвинуть `list1 = list1.next`.
   - Иначе:
     - Прикрепить `list2` к `tail.next`.
     - Сдвинуть `list2 = list2.next`.
   - Сдвинуть `tail = tail.next`.
4. Когда цикл завершится, один из списков может быть непустым. Прикрепить непустой список к `tail.next`: `tail.next = list1 or list2`.
5. Вернуть `dummy.next`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N + M)**, где N и M — длины двух списков. В худшем случае мы пройдем по каждому узлу обоих списков ровно один раз.
- **Сложность по памяти (Space Complexity):** **O(1)**. Мы переиспользуем существующие узлы, просто перекидывая их ссылки (указатели `next`). Дополнительная память выделяется только под фиктивный узел `dummy` и несколько указателей.
