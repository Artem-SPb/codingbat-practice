# [34. Find First and Last Position of Element in Sorted Array](https://leetcode.com/problems/find-first-and-last-position-of-element-in-sorted-array/)

## Описание задачи

Дан массив целых чисел `nums`, отсортированный в порядке неубывания. Необходимо найти начальную и конечную позицию заданного числа `target`.

Если `target` не найден в массиве, верните `[-1, -1]`.

**Строгое ограничение:** Вы должны написать алгоритм со сложностью по времени **O(log n)**.

**Пример 1:**

- **Input:** nums = [5,7,7,8,8,10], target = 8
- **Output:** [3,4]
- **Пояснение:** Число 8 начинается на индексе 3 и заканчивается на индексе 4.

**Пример 2:**

- **Input:** nums = [5,7,7,8,8,10], target = 6
- **Output:** [-1,-1]

**Пример 3:**

- **Input:** nums = [], target = 0
- **Output:** [-1,-1]

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Проблема классического бинарного поиска:** Обычный бинарный поиск останавливается, как только находит `target`. Но если в массиве много восьмерок (например, `[8, 8, 8, 8, 8]`), обычный поиск ткнет в любую из них посередине. Нам же нужны строгие края.
- **Модификация поиска (Поиск границ):** Мы можем запустить бинарный поиск **дважды**.
  1. **Поиск левой границы:** Когда мы находим `target`, мы не останавливаемся! Мы говорим: «Отлично, я нашел 8, но, возможно, слева есть еще восьмерки». Мы запоминаем текущий индекс и сдвигаем правую границу поиска влево (`right = mid - 1`), продолжая искать.
  2. **Поиск правой границы:** Делаем то же самое, но когда находим `target`, сдвигаем левую границу вправо (`left = mid + 1`), чтобы проверить, нет ли еще восьмерок дальше.
- **Переиспользование кода:** Чтобы не писать два цикла `while`, мы вынесем логику поиска в отдельную вспомогательную функцию, которая принимает флаг `find_first` (ищем первую позицию или последнюю).

### 2. Алгоритм

1. Определить вспомогательную функцию `binary_search(nums, target, find_first)`:
   - Инициализировать `left = 0`, `right = len(nums) - 1`, `result = -1`.
   - Пока `left <= right`:
     - Найти середину: `mid = (left + right) // 2`.
     - Если `nums[mid] > target`, ищем слева: `right = mid - 1`.
     - Если `nums[mid] < target`, ищем справа: `left = mid + 1`.
     - Если `nums[mid] == target`:
       - Запомнить индекс: `result = mid`.
       - Если `find_first == True`, продолжаем искать левее: `right = mid - 1`.
       - Иначе (ищем конец), продолжаем искать правее: `left = mid + 1`.
   - Вернуть `result`.
2. В основной функции вызвать `binary_search` дважды: один раз для левой границы, второй раз для правой.
3. Вернуть массив из двух полученных индексов.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(log N)**. Мы запускаем стандартный бинарный поиск ровно два раза. `2 * O(log N)` асимптотически равно `O(log N)`.
- **Сложность по памяти (Space Complexity):** **O(1)**. Мы используем только несколько указателей, дополнительная память не требуется.
