# [11. Container With Most Water](https://leetcode.com/problems/container-with-most-water/)

## Описание задачи

Дан целочисленный массив `height` длины `n`. На координатной плоскости нарисовано `n` вертикальных линий, где каждая линия проведена от точки `(i, 0)` до точки `(i, height[i])`.

Необходимо найти две линии, которые вместе с осью X образуют контейнер, вмещающий наибольшее количество воды. Вернуть максимальное количество воды.

**Ограничение:** Наклонять контейнер нельзя.

**Пример:**

- **Input:** height = [1,8,6,2,5,4,8,3,7]
- **Output:** 49
- **Пояснение:** Максимальное количество воды получается между линиями с индексами 1 (высота 8) и 8 (высота 7). Расстояние между ними = 7. Высота воды ограничена меньшей линией (7). Итоговая площадь: 7 \* 7 = 49.

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Подход "в лоб" (Brute-force):** Можно перебрать все возможные пары линий с помощью вложенных циклов и для каждой пары вычислить площадь. Это потребует проверки `n * (n - 1) / 2` комбинаций, что дает временную сложность O(n²). На больших массивах этот код получит "Time Limit Exceeded".
- **Оптимальный подход (Two Pointers):** Площадь контейнера зависит от двух факторов: расстояния между линиями (ширины) и высоты самой короткой линии (воды не может быть выше короткого борта).

  Мы можем поставить два указателя по краям массива: `left` на начало, `right` на конец. В этой позиции у нас **максимально возможная ширина**.
  Чтобы попытаться найти б*о*льшую площадь, нам нужно сближать указатели. Какой указатель двигать?
  Если мы сдвинем указатель, стоящий на более высокой линии, ширина уменьшится, а высота воды все равно будет ограничена второй (более короткой) линией. Площадь гарантированно станет меньше.
  Следовательно, единственный шанс увеличить площадь — **сдвигать указатель, указывающий на меньшую высоту**, в надежде найти более высокую линию внутри.

### 2. Алгоритм

1. Инициализировать указатели `left = 0` и `right = len(height) - 1`.
2. Завести переменную `max_area = 0` для хранения рекордной площади.
3. Запустить цикл, пока `left < right`:
   - Вычислить текущую ширину: `width = right - left`.
   - Вычислить текущую высоту: `h = min(height[left], height[right])`.
   - Найти текущую площадь: `current_area = width * h`.
   - Обновить `max_area = max(max_area, current_area)`.
   - Сдвинуть указатель с меньшей высотой:
     - Если `height[left] < height[right]`, то `left += 1`.
     - Иначе `right -= 1`.
4. Вернуть `max_area`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(n)**, так как мы используем два сближающихся указателя и проходим массив ровно один раз.
- **Сложность по памяти (Space Complexity):** **O(1)**, поскольку мы не создаем дополнительных структур данных, а используем только несколько переменных.
