# [56. Merge Intervals](https://leetcode.com/problems/merge-intervals/)

## Описание задачи

Дан массив `intervals`, где `intervals[i] = [start, end]` представляет собой интервал. Необходимо объединить все перекрывающиеся интервалы и вернуть массив неперекрывающихся интервалов, которые покрывают все интервалы во входном массиве.

**Пример 1:**

- **Input:** intervals = [[1,3],[2,6],[8,10],[15,18]]
- **Output:** [[1,6],[8,10],[15,18]]
- **Пояснение:** Интервалы [1,3] и [2,6] перекрываются, поэтому они объединяются в [1,6].

**Пример 2:**

- **Input:** intervals = [[1,4],[4,5]]
- **Output:** [[1,5]]
- **Пояснение:** Интервалы [1,4] и [4,5] считаются перекрывающимися.

---

## Мой подход к решению

### 1. Размышления (Thought Process)

- **Проблема хаоса:** Изначально интервалы могут идти в любом порядке (например, `[[15,18], [1,3], [8,10], [2,6]]`). Проверять каждый интервал с каждым — это `O(N^2)`, что слишком медленно.
- **Сортировка решает проблему:** Если мы отсортируем интервалы по времени их _начала_, мы гарантируем, что любой следующий интервал начнется либо одновременно с текущим, либо позже. Это позволяет нам пройти по массиву всего один раз (жадный подход).
- **Логика слияния:** Заведем результирующий список `merged`. Берем очередной интервал и сравниваем его с _последним_ интервалом, добавленным в `merged`:
  1. Если `merged` пуст или конец последнего интервала меньше начала текущего (например, `[1, 3]` и `[8, 10]`), они **не перекрываются**. Просто добавляем текущий интервал в `merged`.
  2. Если они **перекрываются** (например, `[1, 3]` и `[2, 6]`), мы обновляем конец последнего интервала в `merged`. Новым концом станет максимум из двух концов: `max(конец_последнего, конец_текущего)`. _Важно брать именно максимум, так как один интервал может быть полностью внутри другого (например, `[1, 5]` и `[2, 3]`)._

### 2. Алгоритм

1. Если массив `intervals` пуст, вернуть пустой массив.
2. Отсортировать массив `intervals` по первому элементу каждого интервала (по `start`).
3. Создать пустой список `merged`.
4. Запустить цикл по каждому `interval` в отсортированном `intervals`:
   - Если `merged` пуст ИЛИ `merged[-1][1] < interval[0]`: добавляем `interval` в `merged`.
   - Иначе: обновляем конец последнего интервала: `merged[-1][1] = max(merged[-1][1], interval[1])`.
5. Вернуть `merged`.

---

## Анализ сложности

- **Сложность по времени (Time Complexity):** **O(N log N)**, где N — количество интервалов. Сам проход по массиву занимает `O(N)`, но первоначальная сортировка требует `O(N log N)`, что и становится доминирующим фактором.
- **Сложность по памяти (Space Complexity):** **O(N)** в худшем случае (если ни один интервал не перекрывается, массив `merged` будет содержать все N интервалов). Также встроенная сортировка в Python (Timsort) в худшем случае требует `O(N)` дополнительной памяти.
